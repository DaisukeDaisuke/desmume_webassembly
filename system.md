# DeSmuME WebAssembly shared agent context

This file contains durable context that is useful to every agent. Task-specific progress belongs in the relevant live handoff, currently `DEBUGGER_LOGIC_FIX_LIVE_HANDOFF.md`.

## Project shape

- `public/index.html` is the UI shell; application behavior lives in first-party modules under `src/` and is bundled to generated `public/app.js`.
- `public/desmume.js` is generated by Emscripten as a single-file WASM payload. Do not inspect or hand-edit it.
- Native WebAssembly integration is mainly in `webassembly/wasm-port.cpp`; upstream DeSmuME source is the `old/desmume` submodule.
- ROM, save, State, and injected-script data must stay local. Never print their raw contents or derived private data into chat, logs, tests, or public artifacts.

## Durable implementation boundaries

- UI commands and WebMCP calls share the command dispatcher. Background/UI work should use that boundary rather than calling native behavior directly.
- Breakpoints are logical owner-managed sites. User, script, and operation owners may coexist; removing one owner must not remove another. Native breakpoint state must be explicitly reconciled after ROM reload/reset.
- Execute breakpoints stop before the instruction. Safe continuation temporarily disables only the current exec site, steps once, restores the site, and honors any different breakpoint hit during that step. Read/write callbacks must never use exec step-past.
- Persistent scripts run through trusted supervisor Workers and hardened sandbox Workers. User code must have no network, DOM, storage, sub-Worker, raw-message, string-timer, eval, or Function-constructor capability.
- Sandbox `ready` means lockdown is complete. Dynamic import is rejected with a real JavaScript parser both before Worker creation and again inside the sandbox before compilation.
- ROM/save reload is transactional: stage candidates, load and reconcile, then commit metadata. Failure restores the previous files/metadata where possible and remains safely paused.
- Completed native frame count, not attempted calls, is the only source for tick events, audio pumping, frame-step results, and incomplete-frame detection.
- Successful results may be bounded with explicit truncation metadata. Typed error code, message, recoverability, and bounded useful details must not be silently omitted.

## Supply-chain policy

- Production JavaScript dependencies are exact-version npm dependencies installed only in Codespace/CI and bundled locally. Runtime CDN executable code is not allowed.
- Acorn is pinned at `8.17.0` for injected-script AST validation.
- Third-party notices and the lockfile must remain synchronized with every production-bundled dependency.

## Verification and environment

- Do not install npm packages on the Windows host. Use the Codespace for `npm ci`, dependency-backed checks, and production bundling.
- Run only explicit first-party tests; never allow generic discovery to enter external/untracked directories such as `pixelmatch/`, `ssim/`, or vendored/generated trees.
- Browser acceptance uses Chrome DevTools MCP only, never Browser Use. Do not use a headless Chrome substitute.
- Sync source to Codespace with `gh codespace cp -e`. If a Codespace was started, stop it after artifacts and results are recovered.
- Preserve unrelated user changes and untracked files. Authored edits use `apply_patch`.
